# DFS(Depth-First Search)

## 개요

- 그래프나 트리 자료구조를 탐색하는 알고리즘
- 가능한 깊이까지 탐색한 후 더 이상 갈 수 없을 때 되돌아오며 탐색하는 방식
- 그래프의 모든 정점을 방문하는 데 사용될 수 있음 (재귀/스택 구현)
- 방문한 노드 여부 검사 필수 // 무한루프 빠질 위험 존재

    - 시작 정점 선택: 탐색을 시작할 출발점 선택
    - 깊이 우선 탐색: 시작 정점에서 출발해 인접한 정점으로 이동
        - 이동 가능한 정점이 더 이상 없을 때까지 탐색 진행
    - 되돌아오며 탐색: 더 이상 이동할 정점이 없으면 이전 정점으로 되돌아가서 다른 경로 탐색 // 백트래킹
    - 모든 정점 방문: 모든 정점을 방문할 때까지 위 과정 반복 // 완전 탐색

<br>


## 특징

- 시간 복잡도
    - Adjacency list(인접 리스트) :  O(V+E)
        - 각 정점마다 연결된 다른 정점들의 리스트를 저장하는 방식
        - V : 정점의 수, E : 간선의 수입니다.
        - 각 정점을 한 번씩 방문하고 각 간선을 한 번씩 탐색
    - Adjacency Matrix(인접 행렬) : O(V^2)
        - 정점 간의 연결 관계를 2차원 배열로 표현하는 방식
        - V는 정점의 수
        - 모든 정점 간의 연결을 확인 // 각 정점마다 모든 다른 정점들과의 연결 확인
- 

- 공간 복잡도
    - 간선의 수가 적은 희소 그래프(sparse graph)에서는 인접 리스트
    - 간선의 수가 많은 밀집 그래프(dense graph)에서는 인접 행렬이 더 효율적일 수 있음

<br>

## 작동 방식

1. 시작 정점 선택 (0)

```
    0
   / \
  1   2
 /|   |\
3 4   5 6

```

2. 맨 위 정점 0을 가져와 인접 정점 1, 2를 스택에 추가 
    - 방문: [0]
    - 스택: [2, 1]

```
    0*
   / \
  1   2
 /|   |\
3 4   5 6

```

3. 맨 위 정점 1을 가져와 방문하고 인접 정점 3, 4를 스택에 추가
    - 방문: [0, 1]
    - 스택: [2, 4, 3]

```
    0*
   / \
  1*  2
 /|   |\
3 4   5 6

```

4.  맨 위 정점 3을 가져와 방문
    - 방문: [0, 1, 3]
    - 스택: [2, 4]

```
    0*
   / \
  1*  2
 /|   |\
3* 4   5 6

```

5. 정점 3에서 더 이상 이동할 정점이 없어 스택에서 맨 위 정점 4를 가져와 방문
    - 방문: [0, 1, 3, 4]
    - 스택: [2]

```
    0*
   / \
  1*  2
 /|   |\
3* 4*  5 6

```

6. 정점 4에서 더 이상 이동할 정점이 없어 스택에서 맨 위 정점 2를 가져와 방문하고 인접 정점 5, 6을 스택에 추가
    - 방문: [0, 1, 3, 4, 2]
    - 스택: [6, 5]

```
    0*
   / \
  1*  2*
 /|   |\
3* 4*  5 6

```

7. 스택에서 맨 위 정점 5를 가져와 방문
    - 방문: [0, 1, 3, 4, 2, 5]
    - 스택: [6]

```
    0*
   / \
  1*  2*
 /|   |\
3* 4*  5* 6

```

8. 정점 5에서 더 이상 이동할 정점이 없어 스택에서 맨 위 정점 6을 가져와 방문
    - 방문: [0, 1, 3, 4, 2, 5, 6]
    - 스택: []

```
    0*
   / \
  1*  2*
 /|   |\
3* 4*  5* 6*

```

## 예제

### 스택

- 가독성이 떨어짐

```java
import java.util.*;

public class DFS {
    // 그래프를 인접 리스트로 표현
    private Map<Integer, List<Integer>> graph = new HashMap<>();
    // 방문 여부를 기록할 배열
    private boolean[] visited;

    // DFS 함수
    public void dfs(int start) { // 시작 정점을 인자로 받음
        Stack<Integer> stack = new Stack<>();
        stack.push(start);

        while (!stack.isEmpty()) {
            // 스택이 비어있지 않은 동안 반복
            // 스택에서 정점을 하나 꺼내고 아직 방문하지 않은 경우 방문 표시 하고 정점 출력
            int current = stack.pop();
            if (!visited[current]) {
                visited[current] = true;
                System.out.print(current + " ");

                // 인접한 정점을 스택에 추가
                // 인접한 정점들이 역순으로 스택에 추가되어야 DFS의 방문 순서가 올바르게 유지됨
                List<Integer> neighbors = graph.get(current);
                if (neighbors != null) {
                    for (int i = neighbors.size() - 1; i >= 0; i--) {
                        int neighbor = neighbors.get(i);
                        if (!visited[neighbor]) {
                            stack.push(neighbor);
                        }
                    }
                }
            }
        }
    }

    // 그래프에 간선을 추가하는 함수
    public void addEdge(int v, int w) {
        graph.computeIfAbsent(v, k -> new ArrayList<>()).add(w);
        graph.computeIfAbsent(w, k -> new ArrayList<>()).add(v); // 무향 그래프의 경우
    }

    // 메인 함수
    public static void main(String[] args) {
        DFS dfs = new DFS();
        dfs.visited = new boolean[7]; // 정점의 개수에 맞게 배열 초기화

        // 간선 추가
        dfs.addEdge(0, 1);
        dfs.addEdge(0, 2);
        dfs.addEdge(1, 3);
        dfs.addEdge(1, 4);
        dfs.addEdge(2, 5);
        dfs.addEdge(2, 6);

        // DFS 시작
        dfs.dfs(0);
    }
}

```

### 재귀 함수

- 일반적인 코드

```java
import java.util.*;

public class RecursiveDFS {
    // 그래프를 인접 리스트로 표현
    private Map<Integer, List<Integer>> graph = new HashMap<>();
    // 방문 여부를 기록할 배열
    private boolean[] visited;

    // DFS 함수 (재귀)
    private void dfs(int v) {
        visited[v] = true;
        System.out.print(v + " ");

        // 현재 정점의 모든 인접 정점들을 확인
        List<Integer> neighbors = graph.get(v);
        if (neighbors != null) {
            for (int neighbor : neighbors) {
                // 인접 정점이 아직 방문되지 않았다면, 재귀적으로 dfs 호출
                if (!visited[neighbor]) {
                    dfs(neighbor);
                }
            }
        }
    }

    // 그래프에 간선을 추가하는 함수
    public void addEdge(int v, int w) {
        graph.computeIfAbsent(v, k -> new ArrayList<>()).add(w);
        graph.computeIfAbsent(w, k -> new ArrayList<>()).add(v); // 무향 그래프의 경우
    }

    // 메인 함수
    public static void main(String[] args) {
        RecursiveDFS dfs = new RecursiveDFS();
        dfs.visited = new boolean[7]; // 정점의 개수에 맞게 배열 초기화

        // 간선 추가
        dfs.addEdge(0, 1);
        dfs.addEdge(0, 2);
        dfs.addEdge(1, 3);
        dfs.addEdge(1, 4);
        dfs.addEdge(2, 5);
        dfs.addEdge(2, 6);

        // DFS 시작
        dfs.dfs(0);
    }
}
```

- 장점
    - 메모리 효율성
        - 인접 리스트를 사용하는 경우 메모리 사용이 효율적
        - 현재 경로를 저장하기 위해서만 메모리를 사용하므로 경로의 길이가 짧을 때 특히 메모리 사용이 적음
    - 사이클 탐지
        - 그래프에서 사이클이 존재하는지 확인하는 데 사용될 수 있음
    - 백트래킹
        - 특정 시작점에서 목표점까지의 경로를 찾는 데 유용
        - 모든 경로를 탐색하여 최종 목적지까지의 경로를 찾는 데 적합
        - 특정 조건을 만족하는 해를 찾기 위해 다양한 경로를 탐색하고 조건에 맞지 않으면 이전 단계로 돌아가는 문제(ex 퍼즐, 게임 상태 탐색) 효과적

- 단점
    - 무한 루프 위험
        - 사이클이 있는 그래프에서 방문한 정점 추적 x 시 무한 루프에 빠질 수 있음 // 정적 추적하는 배열, 집합 필요
    - 경로 깊이 문제
        - 특정 상황에서 매우 긴 경로 타맥해야 할 경우 스택, 재귀 호출로 인해 메모리 사용 급격히 증가 가능 // 스택 오버플로우 발생 가능
