## 정렬 알고리즘?
- 데이터 요소들을 일정한 순서로 배치하기 위한 방법
- 탐색 속도 향상
    - 정렬된 데이터는 이진 탐색과 같은 효율적인 검색 알고리즘을 사용 가능
- 데이터 분석
    - 통계 분석 및 데이터 시각화에 유리
    - 중복된 데이터를 쉽게 식별하고 제거 가능
- 병합 및 집계
    - 여러 데이터셋을 병합/집계 시 정렬이 필수적

<br>

## 정렬 알고리즘 분류

### 1) 비교 기반 정렬 (Comparison Sort)
- 요소들을 비교하여 정렬
    - 버블 정렬: 인접한 요소들을 비교/교환 후 정렬
    - 삽입 정렬: 요소를 정렬된 부분 배열에 삽입하여 정렬
    - 선택 정렬: 배열에서 가장 작은 요소를 찾아 첫 번째 위치와 교환하고 반복해서 정렬
    - 병합 정렬: 배열을 반으로 나눠 각각 정렬 후 병합
    - 퀵 정렬: 피벗을 기준으로 요소 분할해 정렬
    - 힙 정렬: 힙 자료구조를 이용해 정렬

### 2) 분할 정복 정렬 (Divide and Conquer Sort)
- 문제를 작은 부분으로 나누어 해결하는 알고리즘
    - 병합 정렬
    - 퀵 정렬

### 3) 비교 기반이 아닌 정렬 (Non-Comparison Sort)
- 비교 없이 정렬하는 알고리즘
    - 계수 정렬: 데이터의 크기 범위에 대한 카운팅 배열을 사용하여 정렬하는 알고리즘
    - 기수 정렬: 각 자릿수를 기준으로 정렬하는 알고리즘
    - 버킷 정렬: 데이터를 여러 버킷으로 나누어 각각을 정렬한 후 병합하는 알고리즘

<br>

## 안정 vs 불안정

### 안정 정렬 (Stable Sort)
- 동일한 키를 가진 요소들의 상대적인 순서를 유지하는 정렬 알고리즘
- 정렬 후에도 원래 배열에서 앞에 있던 동일한 키를 가진 요소가 정렬된 배열에서도 앞에 위치

```java
[(3, 'A'), (1, 'B'), (2, 'C'), (3, 'D'), (2, 'E')]
```

- 첫 번째 값: 키,  두 번째 값: 식별자
- 안정 정렬을 사용하면 키 값에 따라 정렬된 후에도 같은 키를 가진 요소들 간의 순서는 유지

```java
[(1, 'B'), (2, 'C'), (2, 'E'), (3, 'A'), (3, 'D')]
```

### 불안정 정렬 (Unstable Sort)
- 동일한 키를 가진 요소들의 상대적인 순서를 유지하지 않는 정렬 알고리즘
- 정렬 후에 원래 배열에서 앞에 있던 동일한 키를 가진 요소가 정렬된 배열에서는 뒤에 위치할 수 있음

```java
[(1, 'B'), (2, 'E'), (2, 'C'), (3, 'D'), (3, 'A')]
```

## 정렬 알고리즘 종류

| 정렬 알고리즘     | 시간 복잡도 (최악) | 시간 복잡도 (평균) | 시간 복잡도 (최선) | 공간 복잡도 | 안정성  | 설명                                                                 |
|-------------------|--------------------|--------------------|-------------------|-------------|--------|--------------------------------------------------------------------|
| 버블 정렬         | O(n^2)             | O(n^2)             | O(n)              | O(1)        | 안정 정렬 | 인접한 두 요소를 비교하여 정렬, 구현이 간단하지만 효율성이 낮음                      |
| 선택 정렬         | O(n^2)             | O(n^2)             | O(n^2)            | O(1)        | 불안정 정렬 | 배열에서 가장 작은 요소를 선택하여 첫 번째 위치와 교환, 간단하지만 비효율적                 |
| 삽입 정렬         | O(n^2)             | O(n^2)             | O(n)              | O(1)        | 안정 정렬 | 정렬된 부분 배열에 새 요소를 삽입하여 정렬, 거의 정렬된 데이터에 효율적                   |
| 퀵 정렬           | O(n^2)             | O(n log n)         | O(n log n)        | O(log n)    | 불안정 정렬 | 피벗을 기준으로 배열을 분할하여 정렬, 평균적으로 매우 빠름                              |
| 병합 정렬         | O(n log n)         | O(n log n)         | O(n log n)        | O(n)        | 안정 정렬 | 배열을 반으로 나누어 각각 정렬한 후 병합, 일관된 시간 복잡도와 안정성 제공                |
| 힙 정렬           | O(n log n)         | O(n log n)         | O(n log n)        | O(1)        | 불안정 정렬 | 힙 자료구조를 사용하여 정렬, 추가 메모리 공간이 거의 필요 없음                           |
| 계수 정렬         | O(n + k)           | O(n + k)           | O(n + k)          | O(n + k)    | 안정 정렬 | 데이터의 크기 범위에 대한 카운팅 배열 사용, 정수 정렬에 매우 효율적                       |
| 기수 정렬         | O(nk)              | O(nk)              | O(nk)             | O(n + k)    | 안정 정렬 | 각 자릿수를 기준으로 정렬, 특정 조건 하에 매우 효율적                                     |
| 쉘 정렬           | O(n^2)             | O(n^1.5)           | O(n log n)        | O(1)        | 불안정 정렬 | 삽입 정렬의 일반화 버전으로, 간격을 두고 떨어진 요소들을 비교하여 정렬, 구현이 간단하고 효율적 |
| 트리 정렬         | O(n log n)         | O(n log n)         | O(n log n)        | O(n)        | 안정 정렬 | 이진 탐색 트리를 사용하여 정렬, 트리의 균형에 따라 성능이 좌우됨                           |
